### 2️⃣ **レイアウト変更（3箇所以上）**

#### 🔄 **レイアウト改善 #1: index.html - カウントダウンセクション**

**何を変更したか？**
```html
【変更前】
<div class="col-3">  ← 常に4列固定

【変更後】
<div class="col-6 col-md-4 col-lg-3">  ← レスポンシブ対応
```

**目的と効果：**
| 画面サイズ | 表示列数 | 改善点 |
|----------|--------|------|
| スマートフォン（≤576px） | **2列** | 小画面での圧迫感を軽減 |
| タブレット（768px～） | **3列** | 適度な情報量 |
| デスクトップ（992px～） | **4列** | 元の見た目を維持 |

**説明例：** 「スマートフォンで見たとき、4列だと窮屈で見にくいので、画面サイズに合わせて自動的に2列→3列→4列に変わるようにしました」

### 🔐 パスワード強度メーター（music-signup-page.html）

**実装概要：**
```javascript
// パスワード入力時のリアルタイム計算
password.addEventListener('input', function() {
    const length = this.value.length;
    
    if (length < 6) {
        strength = '弱';        // 赤色
        progress = 33%;
    } else if (length < 12) {
        strength = '中';        // 橙色
        progress = 66%;
    } else {
        strength = '強';        // 緑色

    }
    
    // ビジュアル更新
    updateStrengthBar(progress, strength);
});
```

**ユーザー側のメリット：**
- ユーザーが安全なパスワード設定を意識できる
- リアルタイムフィードバックでストレスなし
- 登録失敗率を低減

### 📷 画像アップロードプレビュー（music-profile-edit.html）

**実装概要：**
```javascript
// ファイル選択時にプレビュー表示
const fileInput = document.getElementById('avatarUpload');

fileInput.addEventListener('change', function(event) {
    const file = event.target.files[0];
    if (file) {
        // FileReader API でブラウザ内で処理（サーバー通信不要）
        const reader = new FileReader();
        
        reader.onload = function(e) {
            // 選択した画像をプレビュー表示
            document.getElementById('avatarPreview').src = e.target.result;
        };
        
        reader.readAsDataURL(file);
    }
});
```

**技術的ポイント：**
- `FileReader API` を使用してクライアント側で処理
- サーバーとの通信が不要なので高速・セキュア
- 複数の画像フォーマット（JPG、PNG、GIF）に対応可能

> 「このプロジェクトでは、テンプレートのWebサイトをオリジナルサイトに改修しました。
> 
> **最初の課題:** すべてのリンクが '#' でプレースホルダー状態で、実際には遷移していませんでした。
> 
> **解決策:** 7ページのテンプレートを確認して、正しいファイルパスに修正。さらに2ページ（登録・プロフィール編集）を新規作成して、合計9ページにしました。
> 
> **レイアウト改善のこだわり:**
> 
> 1つ目は、ホームページのカウントダウン表示。元々は固定4列だったんですけど、スマートフォンで見たときに窮屈だったので、画面サイズに合わせて2列→3列→4列に自動で変わるようにしました。Bootstrap の col-md-4, col-lg-3 という書き方で実現しています。
> 
> 2つ目は、イベント一覧のフィルター。スクロールしてもフィルターが見えるように、position:sticky を使って固定位置に配置しました。これでユーザーが操作しやすくなります。
> 
> 3つ目は、イベント詳細ページの情報レイアウト。メインと右側のサイドバーに分けて、デスクトップでは2段構成、スマートフォンではスタック表示に自動変更されます。
> 
> **コード品質の改善:**
> 
> もともと7つのHTMLファイルに同じCSSが書かれていたので、共通スタイルシートに統合しました。この結果、色を変更するときは1ファイルを修正するだけで済むようになりました。
> 
> **新機能:**
> 
> 登録ページではパスワード強度メーターを実装。リアルタイムで「弱・中・強」を表示することで、ユーザーに安全なパスワードを促します。
> 
> プロフィール編集ページでは、画像アップロード時にプレビューがすぐに表示される機能を付けました。JavaScript の FileReader API を使ってブラウザ側で処理するので、サーバー通信なしで高速です。」
